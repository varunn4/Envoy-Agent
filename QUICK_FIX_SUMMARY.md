# Quick Fix Guide - Lead Card UI Updates

## Issues Fixed

### ✅ Issue: Leads displayed but action buttons not showing
**Solution**: 
- Moved "View Draft" button and action buttons out of conditional that required draft content
- Now shows **immediately** when lead reaches `drafted` status
- Buttons show pending message while draft is being generated

### ✅ Issue: Card layout still compressed/overlapping
**Solution**:
- Improved flexbox with `flex: 1 1 auto` and `minWidth: 200px` on left section
- Fixed width right section with `flexShrink: 0` prevents badge/score from compressing
- Added proper gap spacing between sections

### ✅ Issue: Draft loading state unclear
**Solution**:
- Added loading indicator in DraftSection when `!isDraftLoading`
- Shows animated spinner with message: "Draft is being generated by AI..."
- Action buttons show "Draft is generating... please wait" message

## Key Changes Made

## Component Flow

```
Lead in "drafted" status
    ↓
Show "▼ View Draft" button immediately
    ↓
If no draft content yet:
  → Show action buttons with "generating..." message
    ↓
If draft content available:
  → Show action buttons active (clickable)
  → Can click View Draft to see/edit content
    ↓
User clicks Accept & Send
  → Show spinner "Sending..."
  → Call API endpoint
  → Status updates to "approved" → "sent"
    OR
User clicks Reject
  → Show spinner "Rejecting..."
  → Call onSkip handler
  → Lead marked as skipped
```

## How to Test

1. **Upload a CSV with leads**
2. **Start a run** - Watch lead cards appear
3. **Wait for leads to reach "Drafted" status**
   - Should see: `▼ View Draft` button + action buttons
   - If draft still generating: See spinner message
4. **Click "▼ View Draft"** to expand and see draft
5. **Click "✏ Edit"** to edit email subject/body
6. **Click "✓ Accept & Send"** to approve and send
   - Should show spinner, then update status to "sent"
7. **Click "✕ Reject"** to skip the lead
   - Should mark as "skipped"
